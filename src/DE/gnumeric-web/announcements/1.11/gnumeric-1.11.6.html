Gnumeric 1.11.6 aka "TBD" is now available.

The Gnumeric Team is pleased to announce the availability of Gnumeric version 1.11.6. This release requires the concurrently released goffice 0.9.6 as well as the recently released libgsf 1.14.24.

This is a bugfix release with a few months' worth of accumulated fixes.

  • Andreas
    • Some documentation updates.
    • Fix implicit ')' for array functions. [Bug 680548].
    • Add FLIP function. [Bug 680511].
    • Allow unordered abscissae in INTERPOLATION. [Bug 680510].
    • Fix INTERPOLATION crash. [Bug 680711].
    • Suppress range markers that should be in an external workbook. [Part of Bug 680718].
    • Fix layering of sheet objects in ODF export and import. [Part of Bug 681009].
    • Export/import opacity of fill-colours to/from ODF. [Part of Bug 681009].
    • Some basic custom-shape support in ODF import.
    • Map captions onto text rectangles on ODF import.
    • Add new autofilter op. [Bug 647401].
    • Provide new saver for uncompressed Gnumeric xml. [Bug 672622].
    • Do not change workbook save info when exporting. [Bug 672323].
    • Fix format selection on export. [Bug 681316].
    • Add Repeat Export menu item. [Bug 594154].
  • Jean
    • Fix component references issues. [Bug 680190].
    • Fix linear averaging interpolation. [Bug 680723][Bug 680918].
    • Support semi-transparent fonts in cells. [Bug 678176].
    • Fix sheet object frame label change. [Bug 681307].
  • Morten
    • Fix xls read crash. [Bug 679992].
    • Dependents code cleanups.
    • Make cells regular dependents.
    • Fix non-linear solver crash. [Bug 680719].
    • Improve inter-workbook recalculations.
